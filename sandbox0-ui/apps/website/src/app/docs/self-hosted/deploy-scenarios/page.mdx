# Deploy Scenarios

Sandbox0 supports multiple deployment scenarios to fit different use cases, from quick evaluation to production multi-cluster setups.

## Single-Cluster Scenarios

### Minimal Mode

<Badge variant="accent">Quick Start</Badge>

The simplest deployment with basic sandbox functionality.

**Components:** `internal-gateway` + `manager`

**Use Case:**
- Quick evaluation and testing
- Development environments
- Basic sandbox execution without persistence

Reference manifest:
- <DocLink href="https://github.com/sandbox0-ai/sandbox0/blob/main/infra-operator/chart/samples/single-cluster/minimal.yaml" newTab>single-cluster/minimal.yaml</DocLink>

### Volumes Mode

<Badge variant="success">Production Ready</Badge>

Adds persistent volume support via storage-proxy.

**Components:** `internal-gateway` + `manager` + `storage-proxy`

**Use Case:**
- Applications requiring persistent data
- File system operations
- Volume snapshots and restoration

Reference manifest:
- <DocLink href="https://github.com/sandbox0-ai/sandbox0/blob/main/infra-operator/chart/samples/single-cluster/volumes.yaml" newTab>single-cluster/volumes.yaml</DocLink>

### Network Policy Mode

<Badge variant="warning">Security</Badge>

Adds network isolation and egress control via netd.

**Components:** `internal-gateway` + `manager` + `netd`

**Use Case:**
- Security-sensitive applications
- Egress traffic control
- Network isolation between sandboxes

Reference manifest:
- <DocLink href="https://github.com/sandbox0-ai/sandbox0/blob/main/infra-operator/chart/samples/single-cluster/network-policy.yaml" newTab>single-cluster/network-policy.yaml</DocLink>

### Full Mode

<Badge variant="success">Complete</Badge>

All data plane services with full feature support.

**Components:** `internal-gateway` + `manager` + `storage-proxy` + `netd`

**Use Case:**
- Production deployments
- Full feature set
- High availability requirements

Reference manifest:
- <DocLink href="https://github.com/sandbox0-ai/sandbox0/blob/main/infra-operator/chart/samples/single-cluster/fullmode.yaml" newTab>single-cluster/fullmode.yaml</DocLink>

## Multi-Cluster Scenarios

Multi-cluster deployments separate control plane and data plane for horizontal scaling.

### Control Plane

Deploy the control plane to manage multiple data plane clusters.

**Components:** `edge-gateway` + `scheduler`

**Requirements:**
- External PostgreSQL (shared with data planes)
- Ingress for external access

Reference manifest:
- <DocLink href="https://github.com/sandbox0-ai/sandbox0/blob/main/infra-operator/chart/samples/multi-cluster/control-plane.yaml" newTab>multi-cluster/control-plane.yaml</DocLink>

### Data Plane

Deploy data planes in different clusters/regions.

**Components:** `internal-gateway` + `manager` + `storage-proxy` + `netd`

**Requirements:**
- Same PostgreSQL as control plane
- Connection to control plane
- Unique cluster ID

Reference manifest:
- <DocLink href="https://github.com/sandbox0-ai/sandbox0/blob/main/infra-operator/chart/samples/multi-cluster/data-plane.yaml" newTab>multi-cluster/data-plane.yaml</DocLink>

## Choosing a Scenario

| Scenario | Volumes | Network Policy | Multi-Cluster | Complexity |
|----------|---------|----------------|---------------|------------|
| Minimal | No | No | No | Low |
| Volumes | Yes | No | No | Medium |
| Network Policy | No | Yes | No | Medium |
| Full | Yes | Yes | No | Medium |
| Multi-cluster | Yes | Yes | Yes | High |

### Recommendations

<Callout variant="info">
**Development/Testing:** Start with Minimal mode, then add features as needed.

**Production:** Use Full mode with external PostgreSQL and S3 for reliability and scalability.

**High Scale:** Deploy multi-cluster with one control plane and multiple data planes across regions.
</Callout>

## Storage Options

### Built-in (Development)

```yaml
storage:
  type: builtin
  builtin:
    enabled: true
    persistence:
      size: "20Gi"
```

<Callout variant="warning">
Built-in storage (RustFS) is for development only. Use S3 or OSS for production.
</Callout>

### AWS S3 (Production)

```yaml
storage:
  type: s3
  s3:
    bucket: sandbox0-prod
    region: us-east-1
    endpoint: https://s3.amazonaws.com
    credentialsSecret:
      name: aws-credentials
      accessKeyKey: accessKeyId
      secretKeyKey: secretAccessKey
```

Create the credentials secret:

```bash
kubectl create secret generic aws-credentials \
  --from-literal=accessKeyId=AKIAIOSFODNN7EXAMPLE \
  --from-literal=secretAccessKey=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY \
  -n sandbox0-system
```

### Aliyun OSS (Production)

```yaml
storage:
  type: oss
  oss:
    bucket: sandbox0-prod
    region: cn-hangzhou
    endpoint: https://oss-cn-hangzhou.aliyuncs.com
    credentialsSecret:
      name: oss-credentials
```

## Registry Options

### Built-in Registry

```yaml
registry:
  provider: builtin
  builtin:
    enabled: true
    persistence:
      size: "20Gi"
```

### AWS ECR

```yaml
registry:
  provider: aws
  aws:
    region: us-east-1
    registryId: "123456789012"
    pullSecret:
      name: ecr-pull-secret
    credentialsSecret:
      name: aws-credentials
```

### Other Providers

Sandbox0 also supports GCP GCR, Azure ACR, and Aliyun ACR. See [Configuration](/docs/self-hosted/configuration) for details.
